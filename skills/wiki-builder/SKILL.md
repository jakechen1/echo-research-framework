# Wiki Builder Skill

Build and manage wiki projects using the Hermes wiki builder on W0. Pages are generated by Gemma 4 on L0 with real PubMed/Semantic Scholar citations.

## When to Use
- User asks to "build wiki pages about X", "create a wiki on X", "research X and write wiki pages"
- User asks about wiki status, progress, or page counts
- User asks to stop, resume, or check a wiki build
- User mentions "wiki", "knowledge base", or "build pages"

## Architecture
- **Wiki location**: `/Users/jakeclaw/wiki/topics/` (1500+ pages, 7 categories)
- **Builder script**: `/Users/jakeclaw/.hermes/scripts/wiki_builder.py`
- **Builder log**: `/Users/jakeclaw/.hermes/logs/wiki-builder.log`
- **Builder state**: `/Users/jakeclaw/.hermes/wiki-builder-state.json`
- **Projects file**: `/Users/jakeclaw/.hermes/wiki-projects.json`
- **LLM**: Gemma 4 on L0 (192.168.68.200:11434), ~1-2 min per page
- **Dashboard**: http://192.168.68.201:3002 (Wiki tab shows live progress)

## Commands

### Start a New Wiki Build
Generate wiki pages from a seed topic. Each page is 800-1500 words with verified citations.

```bash
# Start builder (runs in background, ~1-2 hours for depth 2)
HOME=/Users/jakeclaw /Users/jakeclaw/.hermes-venv/bin/python \
  /Users/jakeclaw/.hermes/scripts/wiki_builder.py start \
  "<seed topic description>" \
  --depth 2 --pages-per-level 15 \
  >> /Users/jakeclaw/.hermes/logs/wiki-builder.log 2>&1 </dev/null &
```

Parameters:
- **seed topic**: Detailed description of what to cover (up to 500 chars)
- **--depth**: 1 (outline only), 2 (+ linked concepts), 3 (deep expansion). Default: 2
- **--pages-per-level**: Pages generated per depth level. Default: 15
- Estimated time: depth 2 x 15 pages = ~30 pages, ~45-60 minutes

### Check Build Status
```bash
HOME=/Users/jakeclaw /Users/jakeclaw/.hermes-venv/bin/python \
  /Users/jakeclaw/.hermes/scripts/wiki_builder.py status
```

Returns JSON with: status (running/completed/stopped), completed_pages count, current_page, seed topic.

### Stop a Running Build
```bash
HOME=/Users/jakeclaw /Users/jakeclaw/.hermes-venv/bin/python \
  /Users/jakeclaw/.hermes/scripts/wiki_builder.py stop
```

### Resume a Stopped Build
```bash
HOME=/Users/jakeclaw /Users/jakeclaw/.hermes-venv/bin/python \
  /Users/jakeclaw/.hermes/scripts/wiki_builder.py resume \
  >> /Users/jakeclaw/.hermes/logs/wiki-builder.log 2>&1 </dev/null &
```

### Check Recent Builder Log
```bash
tail -20 /Users/jakeclaw/.hermes/logs/wiki-builder.log
```

### Count Wiki Pages
```bash
find /Users/jakeclaw/wiki/topics -name "*.md" | wc -l
```

### List Categories
```bash
ls /Users/jakeclaw/wiki/topics/
```

### Search Wiki Pages
```bash
grep -rl "<search term>" /Users/jakeclaw/wiki/topics/ | head -20
```

### Read a Wiki Page
```bash
cat "/Users/jakeclaw/wiki/topics/<category>/<slug>.md"
```

## Project Management

Projects are tracked in `/Users/jakeclaw/.hermes/wiki-projects.json`. The dashboard at http://192.168.68.201:3002 shows all projects with live stats.

To register a build as a project (so the dashboard tracks it):
```bash
python3 -c "
import json
from datetime import datetime
d = json.load(open('/Users/jakeclaw/.hermes/wiki-projects.json'))
proj_id = 'proj_' + str(int(datetime.now().timestamp() * 1000))
d['projects'][proj_id] = {
    'id': proj_id,
    'name': '<Project Name>',
    'prompt': '<seed topic>',
    'depth': 2,
    'pagesPerLevel': 15,
    'status': 'running',
    'createdAt': datetime.now().isoformat(),
    'lastRunAt': datetime.now().isoformat(),
    'hidden': False,
    'completedPages': [],
    'projectDir': 'projects/<slug>',
    'linkExisting': False,
}
d['activeProjectId'] = proj_id
d['selectedIds'] = [proj_id]
json.dump(d, open('/Users/jakeclaw/.hermes/wiki-projects.json', 'w'), indent=2)
print(f'Created project {proj_id}')
"
```

## Workflow Example

When user says "Build me a wiki about CRISPR gene editing":

1. Acknowledge: "Starting a wiki build on CRISPR gene editing. This will generate ~30 pages with verified citations. Estimated time: ~1 hour."
2. Start the builder with appropriate seed prompt
3. Tell user they can check progress on the Dashboard (Wiki tab) or ask you for status
4. When user asks for status, run the status command and report progress

## Notes
- Only ONE build can run at a time (single Ollama context on L0)
- Each page takes ~1-2 min (LLM generation + source validation)
- Pages include YAML frontmatter, [[wikilinks]], and verified PubMed/Wikipedia sources
- The dashboard auto-polls builder status every 10s when running
- Git auto-push syncs wiki to GitHub hourly
